js内存释放

Posted by 荆蛋 on 2017-03-20

堆内存少林寺累死了

对象数据类型或者函数数据类型在定义的时候，首先都会开辟一个堆内存，堆内存有一个引用地址，如果外面有变量等知道了这个地址，我们就说这个内存被占用了，不能被销毁了。

var obj1 = {name:"张三"}; var obj2 = obj1;

var obj1 = {name:"jdy",age:'1'}; var obj2 = obj1; obj1 = null; obj2 = null;

我们想要堆内存销毁/释放，只需要把引用它的变量赋值为null即可。如果当前的堆内存没有被任何东西占用了，那么浏览器会在空闲的时候把它销毁。

栈内存

全局作用域：只有当页面关闭的时候，全局作用域才会销毁
私有的作用域（只有函数执行才会产生私有的作用域），一般情况下，函数执行会形成一个新的私有作用域，当私有作用域中的代码执行完毕后，会被销毁。但是还是存在特殊的情况：

当前私用作用域中的内容被作用域以外的东西占用了，那么当前的这个作用域就不能被销毁了（里面的私有变量也不能被销毁）。
1.函数执行返回了一个引用数据类型值，并且在函数外面被其他东西给接收了，这种情况下一般形成的私有作用域都不会被销毁。
function fn() { var num = 100; return function () { } } var a = fn(); //fn执行形成的这个私有作用域就不能被销毁了
2.在私有作用域中，给dom元素绑定方法，一般情况下，我们的私有作用域都不销毁
(function () { oDiv.onclick = function () { } })();//当前自执行函数形成的这个私有的作用域也不销毁


3.下述这种情况，不会马上销毁，fn返回的函数没有被其他东西占用，但是还需要执行一次，所以暂时不能销毁。当返回的值执行完成后，浏览器会在空闲的时候销毁。
function fn() { var a = 100; return function () { } } fn()();
